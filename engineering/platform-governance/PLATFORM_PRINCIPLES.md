## 3. Architecture Principles

- Core before Industry
- Configuration before Customization
- Business Knowledge before Database
- Reusable before Specific
- Composition before Duplication
- Platform before Project
- Business Language before Technical Language

---

## 4. Development Principles

- Documentation before implementation
- Architecture review before merge
- Feature branches only
- Dry Tech validation before approval
- CI must pass
- No developer reminders inside production UI

---

## 5. Final Principle

MJRH is a universal business operating model expressed as software.
