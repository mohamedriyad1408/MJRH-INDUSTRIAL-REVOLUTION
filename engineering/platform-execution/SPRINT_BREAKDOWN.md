# Sprint Breakdown

## Sprint 1 — Generated Navigation + Permission Mapping

**Objectives:** Build role-aware generated navigation without hardcoding old sidebar.

**Deliverables:** navigation asset contract, generated nav loader, role visibility mapping, Dry Tech nav parity subset.

**Blocked by:** none.

**Produces:** foundation for visible continuity.

**Validation:** Dry Tech nav + newly generated org nav.

## Sprint 2 — Capability Pack Registry Foundation

**Objectives:** Define installable pack manifest and asset application.

**Deliverables:** pack registry model, pack asset contract, capability install validation.

**Blocked by:** Sprint 1 permission/nav concepts.

**Produces:** pack-driven platform services.

**Validation:** install a simple CRM/Reporting nav pack into test org.

## Sprint 3 — Work Order + Task Bridge

**Objectives:** Create non-destructive Order → Work Order → Task compatibility bridge.

**Deliverables:** bridge model, task assignment flow, legacy order compatibility.

**Blocked by:** permission and event foundations.

**Produces:** execution model for all industries.

**Validation:** Dry Tech historical orders unchanged; new work order created.

## Sprint 4 — Generic Work-Area Renderer

**Objectives:** Replace station-specific assumptions with configurable work areas.

**Deliverables:** renderer contract, template station/work-area assets, fallback compatibility.

**Blocked by:** Sprint 3.

**Produces:** staged-work UI reusable by industries.

**Validation:** Dry Tech station parity; non-laundry staged workflow demo.

## Sprint 5 — Reporting/Dashboard Pack

**Objectives:** Convert reports/dashboards into generated assets.

**Deliverables:** report definitions, dashboard widgets, first success dashboard shell.

**Blocked by:** entity registry/report service.

**Produces:** owner/business visibility.

**Validation:** Dry Tech reports + generated org dashboard.

## Sprint 6 — Notification + Customer Experience Packs

**Objectives:** Configure messages/triggers/customer portal behavior.

**Deliverables:** event-triggered templates, customer status model, portal config.

**Blocked by:** event bus, work status model.

**Produces:** customer-facing continuity.

**Validation:** customer portal and notifications in Dry Tech/test org.

## Sprint 7 — Accounting Pack + Financial Transaction Mapping

**Objectives:** Generalize finance without breaking historical journals.

**Deliverables:** transaction mapping, order/work-order finance bridge, reports.

**Blocked by:** Work Order bridge.

**Produces:** universal finance foundation.

**Validation:** Dry Tech journals unchanged; new transaction works.

## Sprint 8 — Business Initialization V2 + Generator Completion

**Objectives:** Owner-first flow generates organization.

**Deliverables:** DNA questions, recommendations, generator execution, first success.

**Blocked by:** packs/generator foundation.

**Produces:** self-service SaaS path.

**Validation:** new organization generated end-to-end.

## Sprint 9 — Dry Tech Gold Standard Parity

**Objectives:** Restore Dry Tech behavior through new architecture.

**Deliverables:** nav/report/workflow/customer/finance parity, data pack plan.

**Blocked by:** Sprints 1–8.

**Produces:** Gold Standard ready for demos/training.

**Validation:** full Business Acceptance Validation.
