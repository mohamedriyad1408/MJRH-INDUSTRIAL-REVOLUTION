# MJRH V4 — Layer 2: Legal Identity Domain Blueprint v2.0

## 1. Sovereignty & Persona
- **Constitutional Anchor:** Every L2 entity (Person, Org) MUST derive its existence from an L1 Identity URN.
- **Sovereign Lock:** Once an L2 Persona is bound to Sovereign Root A, it can NEVER interact with Sovereign Root B.

## 2. Functional Hierarchy
- **Position Definition:** The abstract template of responsibilities (Global).
- **Position Instance:** The unique functional seat anchored to an L1 Node.
- **Reporting Line:** Directed Acyclic Graph (DAG) of Position Instances.

## 3. Temporal Assignments (The Pulse of Authority)
- **Assignments:** The immutable, versioned link between a Person and a Position.
- **Primary Singularity:** One Person, One Organization, One Active Primary Assignment at any microsecond (T).
- **Versioning:** No updates allowed. Changes spawn new UUIDs with parent version references.

## 4. Evidence & Audit
- Every L2 mutation emits a Fact to the L1 Outbox.
- Cross-sovereign lookups are physically impossible via L1 RLS integration.
