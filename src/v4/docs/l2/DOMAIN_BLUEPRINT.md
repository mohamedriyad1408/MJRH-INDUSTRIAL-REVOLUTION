# MJRH V4 — Layer 2 Domain Blueprint v1.1

## 1. Legal Entities (The "Who")
- **Person:** Extension of L1 Human Identity. Owns personal attributes.
- **Organization:** Extension of L1 Sovereign Identity. Owns legal registration.

## 2. Functional Units (The "Where")
- **Department:** Functional container anchored to an L1 Node.
- **Position:** A job blueprint (Position != Person). Every position is unique to a Node.

## 3. The Mandate (The "Authority")
- **Assignment:** A temporal binding of a Person to a Position.
- **Sovereign Lock:** An assignment MUST NOT cross the Sovereign Boundary of the Person's root node in L1.

## 4. Reporting Hierarchy
- **Primary Reporting:** Positions report to other Positions.
- **Functional Reporting:** Matrix relationships for cross-departmental projects.
