# MJRH V4 — Enterprise Operating Model (EOM)
## The Blueprint for a Universal Business Operating System

### 1. Structural Layer (Governance & Identity)
- **Concept:** Org-Nodes (Holdings, Entities, Units).
- **Mechanism:** Recursive Tree with Sovereign Authorities.
- **Rule:** Every object, resource, and event belongs to a Node.

### 2. Functional Layer (The "What")
- **Concept:** Business Capabilities.
- **Mechanism:** Self-contained bundles of logic, data-structures, and events.
- **Scaling:** Activated via Registry.

### 3. Execution Layer (The "How")
- **Concept:** Value Streams & Activities.
- **Mechanism:** State-Machine driven by Event-Sourcing.
- **Resources:** Capacity providers (Human, Machine, Material) assigned to Activities.

### 4. Integrity Layer (Governance)
- **Concept:** Policies & Guardrails.
- **Mechanism:** Blocking/Advisory hooks that validate state transitions.
- **Audit:** Immutable Event Store as the only Source of Truth.

### 5. Insight Layer (The "Why")
- **Concept:** Objectives & Measured Outcomes.
- **Mechanism:** KPIs & OKRs mapped directly to Value Stream events.
