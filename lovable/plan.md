
# خطة التنفيذ — وثيقة متطلبات نظام إدارة المغاسل

سأنفّذ المتطلبات على مراحل قصيرة لضمان الجودة والاختبار. كل مرحلة قابلة للتسليم.

## المرحلة 1 — قاعدة البيانات (Migration واحدة)
- إضافة جدول `tenant_features` (مفاتيح مميزات يفعّلها السوبر أدمن لكل مغسلة).
- إضافة `subscription_fee`, `brand_color`, `logo_url`, `is_active` لجدول `tenants`.
- إضافة `branches` (نقاط التشغيل/الفروع) مع `tenant_id`.
- إضافة `daily_salaries` (الرواتب اليومية للموظفين).
- توسيع `expenses` بـ `category` enum: salaries / rent / water / electricity / other + `monthly_percentage`.
- توسيع `employees` بـ `job_role` enum: ops_manager, cs_manager, cleaning_tech, ironing_tech, packing_tech, driver, receptionist.
- إضافة `task_assignments` (ربط الطلب بالموظف في كل محطة + وقت التحويل).
- توسيع `advance_requests`/`leave_requests` لتشمل أنواع: overtime, prayer, lunch, rest, advance, leave.
- توسيع `orders` بـ `discount_percent`, `urgent_fee_amount`, `payment_method` (cash/instapay/card/transfer), `order_type`.
- إضافة `service_categories` (قائمة كي / تنظيف+كي / أحذية وشنط وخياطة) مع ربطها بـ `service_items`.
- جميع الجداول الجديدة مع GRANT + RLS مقيّد بـ `current_tenant_id()`.

## المرحلة 2 — لوحات الأدوار
- **مالك المغسلة `/dashboard`**: 5 بطاقات طلبات (اليوم/مستعجلة/مُسلّمة/متأخرة/نشطة) + قائمة جانبية (الإيرادات، المصروفات، الرواتب، نقاط التشغيل، طلبات الموظفين، الحضور، المواعيد).
- **مدير التشغيل `/ops`**: بطاقات (اليوم/متأخرة/غدًا/مستعجلة/لم تبدأ/نقد/إنستاباي) + توزيع الأدوار على الفنيين + الفريق النشط + طلبات العاملين.
- **خدمة العملاء `/cs`**: بطاقات (اليوم/متأخرة/قيد المعالجة).
- **السوبر أدمن `/admin`**: إضافة صفحات (تفعيل/تعطيل المميزات، إعادة تعيين كلمة المرور، رسوم الاشتراك، تعديل الهوية البصرية).

## المرحلة 3 — محطات العمل (5 محطات)
- صفحات منفصلة: استلام، تنظيف، كي، تغليف، توصيل.
- لكل محطة: قائمتان (واردة/قيد التنفيذ) + زر "تحويل" يفتح Dialog لاختيار الموظف:
  - استلام/توصيل → سائق
  - تنظيف/كي/تغليف → الفني المناسب
- تسجيل وقت التحويل + الموظف المسؤول في `task_assignments`.

## المرحلة 4 — نموذج "طلب جديد" المحدّث
- أزرار خدمات أساسية تفتح قوائم منسدلة للأصناف.
- حقول: نوع الطلب، طريقة الدفع، رسوم استعجال (مبلغ)، خصم %، ملاحظات.

## المرحلة 5 — الموظفين والعملاء والماليات
- دمج "الفنيين" داخل الموظفين مع دور محدد لكل واحد.
- استيراد عملاء من Excel (الاسم/الهاتف/العنوان من اليمين لليسار).
- صفحة المصروفات: حقول مخصّصة (رواتب/إيجار/مياه/كهرباء) + نسبة شهرية.

## ملاحظات تقنية
- كل التغييرات تحت RLS و`current_tenant_id()` لضمان عزل بيانات كل مغسلة.
- لا تغيير في نظام Multi-tenant القائم — فقط توسيع.
- بعد كل مرحلة سأطلب منك التأكيد قبل الانتقال للتالية.
